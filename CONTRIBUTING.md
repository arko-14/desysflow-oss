# Contributing

## Development Setup

```bash
./letsvibedesign
./letsvibedesign dev
```

## Contribution Workflow

1. Fork the repository and create a feature branch.
2. Keep changes focused and small.
3. Update docs when behavior or interfaces change.
4. Run local checks before opening a PR.
5. Open a pull request with a clear problem statement and test notes.

## Pull Request Guidelines

- Explain what changed and why
- Call out risks or limitations
- Include CLI/API examples if behavior changed
- Keep unrelated formatting-only changes out of functional PRs

## Issues

Use GitHub Issues for bug reports and feature requests.

For bug reports, include:
- steps to reproduce
- expected behavior
- actual behavior
- environment details
